# UI/UX 改善バックログ (手触りの向上)

このドキュメントは、アプリケーションの「手触り」や「使用感」を向上させるための改善項目をまとめたものです。
機能的なバグ修正ではなく、ユーザー体験の質（Quality of Life）を高めるためのタスクをリストアップしています。

## 🔴 高優先度 (High Priority) - ユーザーの混乱を招く要因

### 1. 言語表示の統一 (Language Unification)
- **現状**: アプリケーション内で日本語と英語が混在している。
    - ナビゲーション (`FLOW`, `JOURNAL` 等)
    - 睡眠データ入力 (`poor`, `fair`, `good`)
    - 文言 (`TOTAL CAFFEINE` 等)
- **問題点**: ユーザーに未完成な印象を与え、認知負荷を高める。
- **改善案**:
    - 全て日本語に統一するか、完全な多言語対応を行う。
    - 特に選択肢やラベルなどのユーザーインターフェース要素は日本語化が望ましい。

### 2. 摂取記録のフィードバック即時性 (Instant Feedback)
- **現状**: カフェインを記録した後、ダッシュボードの数値（有効カフェイン量）が即座に変化しない場合がある（消化吸収モデルに基づいているため）。
- **問題点**: ユーザーは「記録ボタンを押したのに反応がない？」と誤解し、不安になる。
- **改善案**:
    - 「吸収中」のステータスを表示する。
    - グラフ上に「今後上昇する予測線」をより明確に強調表示する。
    - 記録直後に「+100mg (吸収中)」のようなトーストやインジケーターを出す。

## 🟡 中優先度 (Medium Priority) - モバイル体験の向上

### 3. ジャーナル削除操作の改善 (Delete Interaction)
- **現状**: 履歴リストの削除ボタンが、ホバーするまで表示されない（`opacity: 0`）。
- **問題点**: モバイルデバイスではホバーがないため、削除操作が発見しにくい、あるいは操作できない。
- **改善案**:
    - 常に薄く表示する（例: `opacity: 0.3`）。
    - モバイル向けにスワイプ削除アクションを導入する。

### 4. 情報設計の視覚的強化 (Visual Polish)
- **現状**: ジャーナル（履歴）リストがテキスト情報中心で、直感的な視認性に欠ける。
- **問題点**: パッと見て「何を飲んだか」が分かりにくい。少し寂しい印象。
- **改善案**:
    - リストアイテムに、摂取した飲料のアイコン（カップや缶など）を表示する。
    - カフェイン量だけでなく、時間軸での相対位置などが分かりやすいデザインにする。

## 🔵 低優先度 (Low Priority) - さらなる洗練 (Delight)

### 5. クイック追加機能 (Quick Add)
- **現状**: 飲料を追加するには「選択」→「確認」の2ステップが必要。
- **問題点**: 頻繁に同じものを飲むユーザーにとっては手数が多く感じる。
- **改善案**:
    - よく飲むアイテムを「お気に入り」としてホームに配置。
    - アイコンの長押し（ロングプレス）やダブルタップで、確認画面をスキップして即時登録できる機能。

### 6. 設定機能の整理 (Settings Handling)
- **現状**: プロフィール画面の設定（ギア）アイコンを押すと「開発中です」というトーストが出る。
- **問題点**: 期待外れ感（Disappointment）を与える。
- **改善案**:
    - 設定項目がまだ無いのであれば、アイコン自体を非表示にする。
    - または、簡易的な設定（テーマ切り替え、データ初期化など）だけでも実装し、機能させる。
