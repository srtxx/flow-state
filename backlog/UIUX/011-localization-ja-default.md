# ローカライゼーション対応 (日本語デフォルト)

**優先度**: 🔴 高
**カテゴリ**: i18n
**見積もり時間**: 3時間

## 問題
- UI内のテキストがハードコードされており、英語と日本語が混在している。
- 将来的な多言語対応が困難。
- ユーザーの要望により日本語をデフォルトにする必要がある。

## 解決策
1. i18nライブラリ（`i18next` + `react-i18next` 推奨）の導入。
2. 翻訳ファイル（`locales/ja/translation.json`, `locales/en/translation.json`）の作成。
3. ハードコードされたテキストを翻訳キーに置き換え。
4. アプリケーションのデフォルト言語を日本語（`ja`）に設定。

## 実装ステップ
1. パッケージインストール: `npm install i18next react-i18next i18next-browser-languagedetector`
2. `src/i18n.ts` 設定ファイルの作成（デフォルト: 'ja'）。
3. `src/main.tsx` で `import './i18n';` を追加。
4. 各コンポーネントのテキストを `t('key')` フックで置換。

## 完了条件
- [ ] 全てのユーザー向けテキストが翻訳ファイル経由で表示される。
- [ ] 初回起動時に日本語で表示される。
- [ ] (オプション) 必要に応じて言語切り替えの基盤ができていること。
