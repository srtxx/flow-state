# チャート摂取マーカー表示不具合の修正

**優先度**: 🔴 高  
**カテゴリ**: Bugfix  
**見積もり時間**: 30分

## 問題
ユーザーが「Quick Add」やカスタム入力で分単位（例: 10:14）で記録したカフェイン摂取が、チャート上にアイコン（☕）として表示されない。

## 原因
RechartsのX軸は30分刻みのカテゴリカルデータとして描画されているが、`ReferenceLine` の `x` 属性に渡される `record.time` が分単位の生データであるため、一致せず描画されない。

## 解決策
チャートコンポーネント内で、受け取った `intakeRecords` の時間を最寄りのX軸の目盛り（00分または30分）に丸める処理を追加する。

## 対象ファイル
- `src/components/AlertnessChart.tsx`

## 完了条件
- [ ] 10:14 などの端数時間で入力しても、チャートの 10:00 または 10:30 の位置にアイコンが表示されること。
