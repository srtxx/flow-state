# ジャーナル履歴の空状態（Empty State）の改善

**優先度**: 🟢 低  
**カテゴリ**: UX Refinement  
**見積もり時間**: 20分

## 問題
現在、ジャーナル（履歴）ページにデータがない場合、シンプルなテキストと追加を促すメッセージが表示されるだけである。ユーザー体験として少し寂しく、アクションへの誘導が弱い。

## 解決策
- イラストやアイコンを大きく使用し、視覚的に訴求するデザインに変更する。
- 「最初のコーヒーを記録しよう」といった、より前向きでアクションを促すコピーに変更する。
- `QuickAdd` へのショートカットボタンを配置することも検討。

## 対象ファイル
- `src/pages/JournalPage.tsx`

## 完了条件
- [ ] 記録ゼロの状態が、ユーザーにとって「記録したくなる」ような魅力的な画面になっていること。
