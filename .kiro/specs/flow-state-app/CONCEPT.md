# Flow State - アプリケーションコンセプト

## アプリケーション概要

**Flow State**は、科学的根拠に基づいたカフェイン摂取最適化・覚醒度追跡アプリケーションです。ユーザーの認知パフォーマンスを最大化しながら、睡眠の質を保護することを目的としています。

**プロジェクト名:** Flow State  
**バージョン:** 1.0  
**ステータス:** 本番稼働中  
**デプロイURL:** https://flow-state-vert.vercel.app  
**最終更新:** 2026-02-09

## ターゲットユーザー

- 認知パフォーマンスを最適化したい知識労働者
- 学習スケジュールと覚醒度を管理する学生
- 一日中集中力を維持する必要がある専門職
- エビデンスベースのカフェイン管理に興味がある全ての人

## コアバリュー

1. **覚醒度の最適化** - インテリジェントなカフェイン推奨による覚醒度の維持
2. **リアルタイム追跡** - カフェイン摂取と覚醒レベルの効果をリアルタイムで追跡
3. **睡眠の保護** - 安全なカットオフタイムの計算による睡眠品質の保護
4. **データ駆動の洞察** - 週次パフォーマンストレンドに基づく洞察の提供

## 主要機能一覧

### 1. ユーザーオンボーディング
- ウェルカム画面でアプリの目的と利点を説明
- 目標睡眠時間の設定（4-10時間の範囲）
- 初回起動時の睡眠時間入力
- オンボーディング完了状態の保存

### 2. 睡眠設定
- 就寝時刻と起床時刻の入力（HH:mm形式）
- 睡眠の質の選択（Poor/Fair/Good）
- 実際の睡眠時間の自動計算
- ローカルストレージへの永続化
- プロフィールページでの表示と編集
- 睡眠データ変更時の覚醒度ベースラインの再計算

### 3. カフェイン摂取記録
- 一般的な飲み物のクイック追加ボタン
  - コーヒー: 100mg
  - エスプレッソ: 80mg
  - エナジードリンク: 80mg
  - 紅茶: 50mg
- カスタムカフェイン量の入力（1-1000mg）
- カスタム時刻の入力（デフォルトは現在時刻）
- 各摂取記録への一意のID生成
- タイムスタンプによる適切な順序付け
- ローカルストレージへの永続化
- ジャーナルページでの記録表示
- 摂取確認前の覚醒度への影響シミュレーション

### 4. 摂取履歴管理
- 当日の全摂取記録を新しい順に表示
- 1日の総カフェイン摂取量の表示
- 各記録の飲み物タイプ、量、時刻の表示
- 確認ダイアログ付きの記録削除機能
- 記録削除時の覚醒度計算の更新
- 記録がない場合の空状態メッセージ
- 1日の推奨上限（FDA推奨400mg）のステータス表示

### 5. リアルタイム覚醒度モニタリング
- 現在の覚醒度スコア表示（0-100スケール）
- ステータスインジケーター（Good/Peak/Low/Over Limit）
- 24時間の覚醒度予測をインタラクティブなチャートで可視化
- ベースライン覚醒度（睡眠ベース）とカフェイン効果の区別
- チャート上の現在時刻マーカー
- チャート上の摂取時刻マーカー
- 時間経過に伴うリアルタイム覚醒度更新
- 摂取記録変更時の予測再計算

### 6. カフェイン推奨
- 適切な場合に現在時刻から30分後に100mgのカフェインを推奨
- 推奨条件:
  - 2時間以内に最近の摂取がない
  - 1日の総摂取量が400mg未満
  - カットオフタイム（睡眠の9時間前）より前
- ダッシュボードでの推奨の目立つ表示
- 推奨から摂取モーダルへのクイックアクション

### 7. 睡眠保護
- 就寝時刻の9時間前を避けるべき時刻として計算
- プロフィールページでの避けるべき時刻の目立つ表示
- 重要性を強調する視覚的強調（ダークカード）
- 睡眠スケジュール変更時の避けるべき時刻の更新
- 避けるべき時刻以降の推奨防止

### 8. 睡眠分析
- 前夜の実際の睡眠時間の表示
- 実際の睡眠と目標睡眠の比較
- 睡眠の質インジケーター
- 睡眠負債の計算と表示（ある場合）
- 重大な睡眠負債に対する視覚的警告
- 睡眠の質が覚醒度に与える影響の説明

### 9. 週次パフォーマンス追跡
- 今週の平均覚醒度の計算
- 先週の平均覚醒度の計算
- トレンドインジケーター付きの週次変化率の表示
- 追跡日数の表示
- 過去14日間の日次スコアの保存
- 現在の覚醒度に基づく日次スコアの自動更新
- データ不足時の「データ不足」メッセージ

### 10. 摂取シミュレーション
- ユーザーが摂取詳細を入力する際の予測覚醒度カーブの表示
- 量または時刻の変更に応じたリアルタイム予測更新
- 実際のデータとシミュレーションデータの視覚的区別
- 確認前のパラメータ調整
- 確認せずにモーダルを閉じた際のシミュレーションクリア

### 11. 多言語サポート
- デフォルト言語として日本語をサポート
- フォールバック言語として英語をサポート
- すべてのUIテキスト、ラベル、メッセージの翻訳
- 言語切り替え（将来の機能拡張）
- 言語設定の永続化

### 12. ダークモードサポート
- システムのダークモード設定の自動検出
- すべてのコンポーネントへのダークカラースキームの適用
- ダークモードでのWCAG AAコントラスト比の維持
- ダークモードでの視認性のためのチャート色の調整
- ダークモードの美観のための影と境界線の調整

## 科学的基盤

### カフェイン代謝モデル
- **半減期:** 5時間
- **ピーク時間:** 45分（0.75時間）
- **吸収フェーズ:** 指数関数的上昇（0-45分）
- **減衰フェーズ:** 指数関数的減衰（45分以降）
- **用量反応曲線:** 飽和効果を持つ曲線

### 覚醒度計算モデル
- **Two-Process Model** (Borbély, 1982)に基づく
  - **Process S:** 恒常性睡眠圧（覚醒時間に応じて蓄積）
  - **Process C:** 概日リズム（24時間の生物時計）
- **睡眠の質修正係数:**
  - Good: 1.0（ペナルティなし）
  - Fair: 0.9（10%減少）
  - Poor: 0.75（25%減少）
- **睡眠負債ペナルティ:** 睡眠不足1時間あたり5ポイント

### 安全ガイドライン
- **1日の上限:** 400mg（FDA推奨）
- **睡眠前のカットオフ:** 9時間
- **推奨間隔:** 最後の摂取から2時間

## データ管理

### ローカルストレージ
- **キー:**
  - `flow-state-sleep`: 睡眠データ
  - `flow-state-intakes`: 摂取記録
  - `flow-state-onboarded`: オンボーディング状態
  - `flow-state-daily-scores`: 日次スコア

### データ保持ポリシー
- 摂取記録: 当日のみ保持
- 日次スコア: 過去14日間保持
- 睡眠データ: 無期限保持

### プライバシー
- すべてのデータはローカルに保存（サーバー送信なし）
- 個人識別情報は収集しない
- 外部APIコールなし（オプションの分析を除く）
