## 設定ファイルの使い方

### 基本的な使用方法

```bash
# 1. デフォルト設定から始める
cd .kiro/specs/document-organization-system/config
cp config.default.json config.json

# 2. 説明フィールドを削除（オプション）
# エディタで *Description フィールドを削除

# 3. システムを実行
npm run organize
```

### カスタム設定を使用

```bash
# カスタム例をベースに作成
cp config.custom-example.json config.json

# 必要に応じて編集
# - includePaths を追加/削除
# - weights を調整
# - thresholds を変更

# システムを実行
npm run organize
```

## 設定の主要なポイント

### 1. Scanner 設定

**includePaths**: スキャン対象のディレクトリ
- デフォルト: `.agent`, `.kiro`, `backlog`, `docs`
- カスタム例: 上記に加えて `documentation`, `wiki`, `notes`, `specs`

**excludePaths**: 除外するディレクトリ
- デフォルト: `node_modules`, `.git`, `dist`, `build`, `.vercel`, `.npm-cache`
- カスタム例: 上記に加えて `coverage`, `tmp`, `temp`, `.next`, `.nuxt`, `vendor`, `archive`

**fileExtensions**: 対象ファイル形式
- デフォルト: `.md`, `.txt`
- カスタム例: `.md`, `.txt`, `.rst`, `.adoc`

### 2. Evaluator 設定

**weights**: 重要度の重み付け（合計 1.0）
- `referenceCount`: 参照数の重要度（デフォルト: 0.4）
- `recency`: 最新性の重要度（デフォルト: 0.3）
- `size`: サイズの重要度（デフォルト: 0.1）
- `category`: カテゴリの重要度（デフォルト: 0.2）

**thresholds**: 判定閾値（0-100）
- `necessary`: この値以上は「必要」（デフォルト: 60）
- `unnecessary`: この値以下は「不要」（デフォルト: 30）
- 中間は「要確認」

### 3. Output 設定

**outputPath**: レポートの出力先
- デフォルト: `DOCUMENT_ORGANIZATION_REPORT.md`
- カスタム例: `reports/document-analysis.md`

**includeMetadata**: メタデータを含めるか（デフォルト: true）

**includeReasoningDetails**: 判断理由を含めるか（デフォルト: true）

## 推奨される使用シナリオ

### config.default.json を使用
- 初めてシステムを使用する場合
- 標準的なプロジェクト構成の場合
- 設定の意味を理解したい場合

### config.custom-example.json を使用
- 大規模プロジェクトの場合
- 厳格なドキュメント管理が必要な場合
- 複数のドキュメント形式を扱う場合
- 参照関係を重視する場合

### config.minimal.json を使用
- シンプルな設定が好みの場合
- 小規模プロジェクトの場合
- すぐに使い始めたい場合
